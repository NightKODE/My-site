Lesson 1
Самое важное в JavaScript.

L1. Самое важное в JavaScript.

  • 1. Выражения
  • 2. Функции
  • 3. Объекты
Объект находится в фигурных скобках, а внутри него пары: ключ и его значение. Ключи являются свойствами этого объекта.
Иногда у свойств значением может быть объект, такие объекты называются вложенными.
Число и строка лишь ведут себя как объекты, так как значение типа число или строка - это примитивное значение.
console.log("Hello World")
  • console - объект
  • log - метод
  • Метод - функция, которая является значением одного из свойств объекта.
  • Точечная запись - синтаксис Javascript, с помощью нее можно получать доступ к свойствам объекта.
  • Значения типа строка String помещаются либо в одинарные, либо в двойные кавычки.
  • console.dir() - отображает все свойства объекта
  • console.table() - отображает все свойства объекта в табличном виде
Lesson 2
Выражения.

L2. Выражения.

Любое выражение всегда возвращает значение.
  • Выражение присваивания. =

    Выражение присваивания - это выражение в котором значение (может быть любым) присваивается переменной.

  • Выражения с побочными действиями.

    Они не только возвращают значение, но и выполняют другие действия.

На примере оператора присвоения а = 5 - это выражение не только вернет значение 5, но и выполнит действие присвоения переменной а этого значения.
На примере выражения b++ - если переменной b было присвоено значение 6, то это выражение вернет значение 7, а еще и увеличит значение переменной на 1, так как оператор ++ увеличивает значение переменной на 1.
Lesson 3
Переменные.

L3. Переменные.

Переменные дают возможность повторного доступа к значениям, которые присвоены к этим переменным.
Как правильно называть переменные.
Имена переменных можно выбирать любое, но лучше использовать следующие три варианта:
  • 1. PascalCase - (первое слово названия переменной начинается с заглавной буквы, как и второе слово. Слова пишутся слитно) используется чтобы называть типы и классы в JS.
  • 2. DB_PASSWORD - значение такой переменной известно до запуска приложения и не меняются. (все слова пишутся капсом и между словами используется нижнее подчеркивание.)
  • 3. сamelCase - используется для всех остальных переменных, они используются чаще всего. (первое слово всегда в нижнем регистре, следующее слово с заглавной буквы, слова пишутся слитно)
Название переменной должно быть понятным и отображать значение переменной.
Объявление переменных.
В JS есть три ключевых слова, с помощью которых объявляются переменные:
  • 1. let - let a объявление переменной с названием а, с помощью let мы даем инструкцию интерпритатору JS создать новую переменную с именем а. Присвоить значение этой переменной можно выражением присваивания.

    При использовании let значение переменной можно присвоить сразу, а позже можно присвоить другое значение.

  • 2. const - const c = 10 это объявление переменной с именем с и присваивание этой переменной значения 10 с помощью оператора присвоения.

    При использовании const присвоить другое значение переменной нельзя.

  • 3. var – не рекомендуется использовать
Правила работы с переменными.
  • 1. Все переменные необходимо объявлять перед их использованием.
  • 2. При объявлении переменных использовать const везде, где только возможно.
  • 3. Внутри функции не изменять переменные из внешних областей видимости.

    Для этого лучше объявлять переменные внутри функции.

Строгий режим.
Чтобы избежать автоматического создания переменной в глобальной области видимости, необходимо использовать строгий режим.
Строгий режим - это инстрункция интрепритатора JavaScript анализировать код более пристально.
Чтобы включить строгий режим необходимо в начале файла написать 'use strict'.
В таком случае строгий режим просто не даст использовать перменную без ее объявления и будет выдавать ошибку.

Lesson 4
Типы переменных.

L4. Типы переменных.

Примитивные типы.
  • string (строка) - Значением этой переменной является строка. Значение принято записывать в кавычки (можно в одинарные, можно в двойные, но важно выбрать один тип на весь файл JS).

    Можно использовать обратные ковычки ``(бэктики), однако использование этих ковычек позволяет встраивать в такую строку другие сущности в специальной конструкции ${}

    Эта механика называется интерполяция строк.

  • boolean (логический) - имеет два значение: true либо false
  • number (номер) - этому типу переменных присваивается числовое значение.
  • null - имеет значение null. Этот тип используется тогда, когда необходимо указать, что у какой-то переменной на данном этапе нет значения.
  • undefiend - означает, что у данной переменной нет никакого значения. Присваивается автоматически при объявлении переменной без присвоения значения.
  • symbol (символ) - с помощью него можно создавать уникальные значения. Используется редко.
Ссылочный тип.
Ссылочным типом значения переменной является какой-либо объект.
При присваивании переменной значение объект, переменная не хранит сам объект, она хранит только ссылку на объект.
Lesson 5
Динамическая типизация.

L5. Динамическая типизация.

JavaScript - динамически типизируемый язык, что позволяет присваивать одной переменной разные типи значения при необходимости.
Иногда бывает такое, что была объявлена функция с именем, а затем с этим же именем была объявлена переменная с другим типом значения, что в результате выдает ошибку и ломает функцию. Исправить это можно при помощи объявления переменной словом const.
Lesson 6
Объекты.

L6. Объекты.

Главная идея JavaScript - практически все сущности - объекты.
Объект - набор свойств или пар ИМЯ: ЗНАЧЕНИЕ.(имена часто называют ключами).
Объект находится в фигурных скобках, а внутри него пары: ключ и его значение. Ключи являются свойствами этого объекта.
Порядок свойств в объекте не имеет значения.
Используя точечную запись, можно менять значения свойств объекта и присваивать новые свойства.
Глобальные объекты.
Глобальный объект в JavaScript – это специальный "ящик" 📦, где хранятся переменные и функции, доступные везде в вашем коде.
Есть два глобальных объекта - window(браузер) и global(node.js). Так же можно написать globalThis в обоих местах.

Операторы.

L7. Оператор удаления.

Оператор - это встроенная в JavaScript Функция.
  • Операторы быают: арифметические - + - * /
  • Сравнения - === !== <=< /span> >=
  • Логические - ! (не) && (и) || (или)
  • Опертор присваивания - =
  • Опертор разделения объекта на свойства - ...
Текстовые операторы
  • typeof - С помощью этого оператора можно проверить тип того или иного значения.
  • instanceof - С помощью этого оператора можно проверить принадлежность объекта к тому или иному классу.
  • new
  • delete - Оператором удаления можно удалить то или иное свойство у объекта.
Операнды - это то, что находится справа или слева от оператора.
Унарные и бинарные операторы
У унарных операторов всегда один операнд (аргумент).
++ + delete typeof new Object
У бинарных операторов два операнда (аргумента).
= + += === &&.
Форматы записи операторов
Инфиксная запись - в таком формате операнд находится между операндами.
Префиксная запись - в таком формате оператор стоит перед операндом. При этом у этого оператора имеется возможность постфиксной записи.
Постфиксная запись - в таком формате оператор идет после операнда.

Полностью знать приоритетность операторов не нужно, достаточно просто посмотреть на таблицу в случае необходимости.

Lesson 11
JSON – Javascript object notation.

L11. JSON – Javascript object notation.

JSON - формат обмена данными между компьютерами.
Cвойства записаны в двойных кавычах, так же в двойных кавычках записаны значения этих свойств.
Передача данных в формате JSON происходит по средству строки.
Конвертация JSON.
  • Получив JSON от сервера, его можно передать его как аргумент в вызове метода "parse" JSON.parce( ), и в результате получится JavaScript объект. “Распарсить”
  • Если же необходимо переконвертировать файл JavaScript, применяется метод "stringify" JSON.stringify( ) .
Lesson 12
Мутация в JavaScript.

L12. Мутация в JavaScript.

При примитивных значениях переменных, копирование происходит по принципу copy by Value, то есть, копируется непосредственно само значение.
При значениях типа объект, копирование происходит по принципу copy by Referens, то есть, копируется ссылка на объект, что позволяет изменять (мутировать) объект.
Избежание мутаций.
  • Первый способ:

    Использовать метод assign объекта Object (этот объект является частью глобального объекта. Он так же является и типом и классом в JavaScript).

    При помощи такого синтаксиса можно создать новый объект на основе уже существующего. Таким образом создается новый объект с такими же свойствами, как у предыдущего, но с другой ссылкой на этот объект.

    Если же в предыдущем объекте имеются вложенные объекты, то ссылки на них сохранятся! Таким образом можно избежать мутации только корневых свойств.

  • Второй способ:

    Использовать оператор ..., он же оператор sprat (оператор разделения). Этот оператор разделяет объект на свойства, а записанный вместе с ним объект в фигурных скобках создает новую ссылку на объект.

    В это случае ситуация с вложенными объектами такая же, как и в первом.

  • Третий способ:

    Можно использовать двойную конвертацию объекта через JSON. Сначала объект конвертируется в строку, а потом конвертируется обратно в объект, создавая новые ссылки даже на вложенные объекты.

Lesson 13
Функции.

L13. Функции.

Функция - блок кода, который можно выполнять многократно.
Если в коде часто встречаются одни и те же блоки кода, то такой код написан неоптимально. Чтобы избежать повторения, можно использовать функцию.
После строки возвращения результата фукция прекращает работу.
Функция возвращает undefiend, если нет инструкции return
Вызов функции
При объявлении функции будет создана новая переменная, значением котрой является функция, после чего ее можно вызвать.
При объявлении функции в скобках находятся парметры, являющиеся переменными, которые действуют в функции.
При вызове функции в тех же скобках указываются уже аргументы. Они мгут меняться каждый раз при вызове этой функции.
  • 1. Параметрам а и b присваиваются значения 10 и 3.
  • 2. Объявляется переменная с.
  • 3. Значение а увеличивается на 1.
  • 4. Сумма значений а и b присваевается с.
  • 5. Возвращается значение с.
Передача значения по ссылке
В данном примере происходит мутация объекта personOne, поскольку при указании его в аргументе функции, происходит передача значения по ссылке, а не создается новый объект.
Внутри функции мутировать внешние объекты не рекомендуется. Чтобы этого избежать, можно внутри функции создать копию объекта.
Колбэк функции
Callback function - это функция, короая вызывается внутри другой функции.
setTimeout - это встроенная функция глобадьного объекта Objekt, и означет она задержку перед выполнением функции.
Правила работы с функциями.
  • 1. Называть функции исходя их из выполняемых задач.
  • 2. Одна функция должна выполнять одну задачу.
  • 3. Не рекомендуется изменять внешние относительно функции переменные.

    Лучше создавать копии объектов внутри функции.

Lesson 14
Области видимости.

L1. Области видимости

Обдасть видимости определяется границы действия переменной.
Есть глобальные переменные, которые объявлены в глобальной обласьт видимости, а есть локальный переменные, которые объявлены в локальной области видимости. Если попробовать получить лоступ к переменной b в локальной области видимости 1, то мы увидим результат переменной b, объявленной в глобальной области видимости.
В данном примере функция является локальной областью видимости, в которой была создана еще одна переменная с именем b. Так же внутри этой функции было присвоено значение переменной a = true, однако присвоено оно пременной a из глобальной области видимости.
В данном примере создается переменная myFn, внутри нее создается перменная innerFn, и если попытаться вызвать функцию innerFn, то произойдет ошибка, т.к. innerFn существиет только внутри функции myFn.
Если в локальной области видимости присваяивать значения необъявленным переменным, то по завершению кода эти переменные будут созданы овтоматически. Так делать крайне не рекомендуется.
